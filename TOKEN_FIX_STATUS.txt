🔧 GitHub Token Troubleshooting Guide - แก้ปัญหา Error 403

╔═══════════════════════════════════════════════════════════════════════════════╗
║                    ✅ ปัญหาได้รับการแก้ไขแล้ว - โปรดทดสอบ                      ║
╚═══════════════════════════════════════════════════════════════════════════════╝

🎯 การแก้ไขที่ทำไปแล้ว:

1. ✅ อัปเดต Authorization Header:
   เปลี่ยนจาก: 'Authorization': f'token {token}'
   เป็น:      'Authorization': f'Bearer {token}'

2. ✅ เพิ่ม GitHub API Version Header:
   เพิ่ม: 'X-GitHub-Api-Version': '2022-11-28'

3. ✅ ปรับปรุงข้อความช่วยเหลือ Token:
   เพิ่มรายละเอียด Scopes ที่จำเป็น

╔═══════════════════════════════════════════════════════════════════════════════╗
║                              🧪 วิธีทดสอบ                                     ║
╚═══════════════════════════════════════════════════════════════════════════════╝

1. 🚀 เปิดโปรแกรม Repo Manager
2. 🔍 กด "ตรวจสอบ" เพื่อตรวจสอบ Token
3. 🎯 ลองสร้าง Release ใหม่ด้วย "สร้าง Release"

╔═══════════════════════════════════════════════════════════════════════════════╗
║                          📋 หาก Error ยังเกิดขึ้น                              ║
╚═══════════════════════════════════════════════════════════════════════════════╝

📝 ตรวจสอบ Token Scopes:
   ไปที่: https://github.com/settings/tokens
   ✅ repo (Full control of repositories)
   ✅ workflow (Update GitHub Action workflows)

🔄 สร้าง Token ใหม่หากจำเป็น:
   1. ไปที่ https://github.com/settings/tokens/new
   2. ชื่อ: "Repo Manager Full Access"
   3. Expiration: 90 days
   4. Scopes: repo, workflow
   5. Generate และ Copy token
   6. อัปเดตในไฟล์ .env
