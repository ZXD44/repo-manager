# ?? Repo Manager - ตัวจัดการ GitHub Repository และ Release

`Repo Manager` เป็นเครื่องมือ GUI สำหรับเดสก์ท็อปที่สร้างด้วย Python (Tkinter) เพื่อช่วยให้การจัดการ GitHub repository ของคุณง่ายขึ้น สามารถ Push ไฟล์และสร้าง Release ใหม่ได้อย่างสะดวกสบายผ่านหน้าต่างโปรแกรมที่สวยงามและใช้งานง่าย


## ? คุณสมบัติหลัก (Features)

- **อัปโหลดไฟล์ (Push):** Push การเปลี่ยนแปลงจากเครื่องของคุณไปยัง GitHub repository
- **สร้าง Release:** สร้าง GitHub Release ใหม่ พร้อมระบบแนะนำเวอร์ชันถัดไปและสร้าง Changelog (รายการเปลี่ยนแปลง) จาก commit ล่าสุดโดยอัตโนมัติ
- **ทำทั้งหมด (Do All):** ฟังก์ชันที่รวมการ Push ไฟล์และการสร้าง Release ไว้ในขั้นตอนเดียว
- **ตรวจจับข้อมูลอัตโนมัติ:** อ่านข้อมูลเจ้าของ (Owner) และชื่อ Repository จาก `git remote` ในโปรเจคของคุณ
- **สร้าง Repository ใหม่:** หาก Repository ยังไม่มีบน GitHub สามารถสร้างให้ใหม่ได้โดยอัตโนมัติ (เป็น Public)
- **ดีไซน์สวยงาม:** มาพร้อมธีมสีเข้มสไตล์ GitHub ที่ทันสมัย
- **บันทึกการทำงาน (Logging):** แสดงผลการทำงานทุกขั้นตอนแบบ Real-time
- **ปลอดภัย:** ใช้ GitHub Personal Access Token (PAT) ในการเชื่อมต่อกับ API และไม่ได้บันทึก Token ของคุณไว้ที่ใด

## ?? สิ่งที่ต้องมี (Requirements)

- Python 3.x
- ติดตั้ง Git บนเครื่องคอมพิวเตอร์ของคุณ
- ไลบรารี `requests` (สคริปต์จะพยายามติดตั้งให้โดยอัตโนมัติเมื่อรันครั้งแรก)

## ??? การติดตั้งและเริ่มต้นใช้งาน (Installation & Usage)

1.  ดาวน์โหลดไฟล์ `repo_manager.py`
2.  เปิด Terminal หรือ Command Prompt ในโฟลเดอร์ที่เก็บไฟล์
3.  รันสคริปต์ด้วยคำสั่ง:
    ```bash
    python repo_manager.py
    ```
4.  หากยังไม่มีไลบรารี `requests` สคริปต์จะทำการติดตั้งให้ก่อนเริ่มโปรแกรม

## ?? วิธีใช้งาน (How to Use)

1.  **โฟลเดอร์โปรเจค:**
    - โปรแกรมจะใช้โฟลเดอร์ปัจจุบันเป็นค่าเริ่มต้น
    - สามารถกดปุ่ม **"เลือก"** เพื่อเปลี่ยนไปยังโฟลเดอร์โปรเจค Git ของคุณ
    - โปรแกรมจะพยายามดึงข้อมูล repo จาก remote `origin` โดยอัตโนมัติ

2.  **GitHub Token:**
    - คุณจำเป็นต้องใช้ Personal Access Token (PAT) เพื่อให้โปรแกรมสามารถสื่อสารกับ GitHub ได้
    - กดปุ่ม **"ช่วยเหลือ"** เพื่อดูขั้นตอนการสร้าง Token
    - นำ Token ที่ได้มาวางในช่อง **"GitHub Token"**

3.  **ข้อมูล Repository:**
    - ชื่อ **"เจ้าของ"** และ **"ชื่อ Repository"** จะถูกเติมให้โดยอัตโนมัติ หากไม่ถูกต้อง สามารถแก้ไขได้ด้วยตนเอง

4.  **การดำเนินการ (Actions):**
    - **?? อัปโหลดไฟล์:** ทำการ `git add .`, `git commit`, และ `git push` ไปยัง branch หลัก (`main`)
    - **?? สร้าง Release:** เปิดหน้าต่างสำหรับสร้าง Release ใหม่ โดยจะแนะนำเวอร์ชันและสร้าง Changelog ให้โดยอัตโนมัติ คุณสามารถแก้ไขรายละเอียดได้ก่อนยืนยัน
    - **?? ทำทั้งหมด:** เป็นกระบวนการ 2 ขั้นตอน คือ "อัปโหลดไฟล์" ก่อน จากนั้นจะเปิดหน้าต่าง "สร้าง Release" ต่อเนื่องกัน
    - **?? ตรวจสอบ:** เช็คความพร้อมใช้งานของ Git, Token และ Repository
